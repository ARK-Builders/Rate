# Agent Instructions

- Before working on this project, read both documentation sources: `README.md` and every `docs/*.md` file.
- Keep `README.md` lightweight and targeted at power users who may build the project but are unlikely to contribute.
- Prefer adding or updating detailed development, workflow, and release documentation in `docs/` instead of expanding `README.md`.
- Keep documentation focused on durable, essential information and keep it in sync with the codebase and with `README.md`.
- Treat `README.md` as a concise documentation index. When information is intentionally duplicated between `README.md` and `docs/` because it is useful to both developers and power users, keep the duplicated sections consistent.
- Keep documentation in `docs/` de-duplicated: no two docs files should repeat the same information. Link to another docs file when that keeps the explanation concise and clear without duplicating content.
- For non-trivial design decisions, suggest documenting the decision in `docs/`.
